# Phase 3: Voice Dispatch Tool Architecture ($0 Setup)

This phase adds a conversational AI voice agent to the existing Express.js backend. To keep the setup **100% free**, we bypass paid telephony (Plivo/Twilio) and simulate the "phone call" using **browser-based WebRTC** via LiveKit Cloud's free tier. The underlying AI logic remains identical to what a production phone-call system would use.

## 1. Core Objectives

- Create a conversational AI agent that briefs a worker about a job and collects an Accept/Decline decision.
- Support Indian regional languages (Hindi, Marathi, English, code-mixed).
- Inject dynamic "memory" context (worker name, job `category`, `description`, `address`, `paymentOffer`) from MongoDB into the conversation.
- Update the existing `Job` model's `status` field (`assigned` → `accepted` / back to `pending`) based on the extracted decision.

---

## 2. Tool Stack & Responsibilities

### A. LiveKit Cloud (Free Tier)

**Role: The Audio Bridge**

- Real-time WebRTC communication server.
- Handles audio routing, buffering, and Voice Activity Detection (knows when the user stops speaking so the AI can reply).
- No SIP trunk or PSTN needed — purely WebRTC room connections.

### B. LiveKit Web SDK (Frontend Simulator)

**Role: The "Feature Phone"**

- A simple browser page (can live under `frontend/` or standalone HTML) that replaces a physical phone.
- Uses the laptop microphone and speakers to join a LiveKit room via a token generated by Express.
- Simulates the experience of picking up and answering a call.

### C. Node.js Voice Agent (`@livekit/agents`)

**Role: The Brain / Orchestrator**

- A process (can be the same Express server or a standalone script) that listens for new LiveKit rooms.
- When the simulator connects, the agent joins the same room.
- Fetches "memory" from MongoDB:
  - **Worker context:** `User` doc (`name`, `phone`, `userType`, `status`, `categories`) + assigned `Job` doc (`category`, `description`, `address`, `paymentOffer`, `distance`).
  - **Client context:** `User` doc → used by `categoryExtractor.js` and `matching.controller.js` to create a job and find a worker.
- Wires room audio through STT → LLM → TTS pipeline.
- Parses the final conversation for ACCEPT / DECLINE intent and calls existing controller logic (`acceptJob` / `cancelJob` in `job.controller.js`) to update MongoDB.

### D. Sarvam AI (Hackathon / Free Credits)

**Role: The Ears and Mouth (Speech Layer)**

- **Sarvam STT (`saaras:v1`):** Converts incoming room audio (Hindi, Marathi, English, code-mixed) into text.
- **Sarvam TTS (`bulbul:v1`):** Converts LLM-generated text into natural-sounding audio in the worker's language.
- **Sarvam Chat (`sarvam-2b`):** Lightweight intent extraction — mirrors the logic already used in `categoryExtractor.js` (keyword match → Sarvam AI fallback). Categories supported: `plumber`, `electrician`, `carpenter`, `painter`, `cleaner`, `mechanic`, `ac_technician`, `pest_control`.

### E. OpenAI / Gemini (Free Tier / Credits)

**Role: The Conversational Logic**

- Receives the transcribed text from Sarvam STT.
- Uses a strict System Prompt injected with MongoDB "memory" (job details, worker name, address, payment).
- Decides how to respond: answer questions about the job, re-state the address, or ask for confirmation.
- Outputs text → sent to Sarvam TTS for audio synthesis.

### F. MongoDB (Existing Backend — Atlas)

**Role: The State Manager**

Already in place. Relevant models and their fields used by the voice agent:

| Model | Key Fields for Voice |
|-------|---------------------|
| `User` | `name`, `phone`, `userType` (client/worker), `location` (GeoJSON 2dsphere), `categories[]`, `status` (available/busy/offline) |
| `Job` | `jobId`, `clientId`, `workerId`, `category`, `description`, `paymentOffer`, `location` (GeoJSON), `address`, `status` (pending → assigned → accepted → in_progress → completed/cancelled), `distance` |
| `Category` | `name`, `displayName`, `keywords[]` (used by `categoryExtractor.js`) |

Existing backend logic the voice agent reuses directly:

| Module | What it does |
|--------|--------------|
| `matching.controller.js` → `findWorker` | Scores workers by **50% distance + 30% rating + 20% visibility**, returns ranked match |
| `job.controller.js` → `createJob` | Creates a job, auto-extracts category via `categoryExtractor.js` |
| `job.controller.js` → `acceptJob` | Sets `job.status = 'accepted'`, `job.acceptedAt = Date.now()`, worker `status = 'busy'` |
| `job.controller.js` → `cancelJob` | Resets job back to `pending`, worker `status = 'available'` |
| `utils/categoryExtractor.js` | Hindi keyword match first → Sarvam AI NLP fallback for category detection |
| `utils/distance.js` | Haversine formula (returns km) |

## 3. Step-by-Step Architecture Flow

```text
+---------------------+   WebRTC    +------------------+
| Browser Simulator   | <=========> |  LiveKit Cloud   |
| (Mic + Speaker)     |             |  (Free Tier)     |
+---------------------+             +--------+---------+
                                             |
                                       WebRTC (Agent)
                                             |
                                             v
                                    +------------------+
                                    |   Express.js     |
                                    |   Voice Agent    |
                                    +--------+---------+
                                             |
                              +--------------+--------------+
                              |              |              |
                         Sarvam STT    OpenAI/Gemini   Sarvam TTS
                         (Ears)        (Brain)         (Mouth)
                              |              |              |
                              +--------------+--------------+
                                             |
                                          MongoDB
                                    (User, Job, Category)
```

1. **Trigger:** Express generates a LiveKit Room Token for a specific job via `POST /api/voice/sessions/initiate`. The room name encodes the context (e.g., `job-<jobId>-worker-<userId>`).
2. **Connect:** The developer opens the browser "Worker Phone Simulator" page, which uses the token to activate the laptop mic and join the LiveKit room.
3. **Context Load:** The Node.js Voice Agent detects the new participant, queries MongoDB for the `Job` doc (populating `clientId` for client name) and the `User` doc (worker name, preferred language). Injects `category`, `description`, `address`, `paymentOffer`, `distance` into the LLM system prompt.
4. **Speak:** The agent uses Sarvam TTS to greet the worker in Hindi/English: *"Namaste [workerName], aapke paas ek [category] ka kaam hai, [distance] km door, [paymentOffer] rupees ka. Accept karoge?"*
5. **Listen & Converse:** Worker speaks → Sarvam STT transcribes → LLM processes (answers questions about address/payment, asks for confirmation) → Sarvam TTS speaks reply. Loop continues until intent is clear.
6. **Resolve:** Once the LLM detects a clear Accept or Decline:
   - **Accept:** Agent calls `acceptJob` logic → `job.status = 'accepted'`, `worker.status = 'busy'`, `job.acceptedAt` stamped.
   - **Decline:** Agent calls `cancelJob` logic → `job.status = 'pending'`, `worker.status = 'available'`, job re-enters the matching queue.
   - Agent speaks confirmation, then disconnects from the LiveKit room (ending the "call").

```javascript
// src/routes/voice.routes.js
import express from 'express';
import { authenticate } from '../middleware/auth.middleware.js';
import livekitService from '../services/livekit.service.js';
import voiceAgentService from '../services/voiceAgent.service.js';

const router = express.Router();

// Generate a LiveKit room token for the browser simulator
router.post('/sessions/initiate', authenticate, async (req, res) => {
  const { jobId, targetUserId } = req.body;
  const roomName = `job-${jobId}-user-${targetUserId}`;
  const token = await livekitService.createRoomToken(roomName, `simulator-${targetUserId}`);

  // Start the AI agent in the background (do not await)
  voiceAgentService.startVoiceSession(roomName, targetUserId, jobId).catch(console.error);

  res.json({ roomName, token, livekitUrl: process.env.LIVEKIT_URL });
});

export default router;
```

## 5. Service Implementations

### A. livekit.service.js

```javascript
// src/services/livekit.service.js
import { AccessToken } from 'livekit-server-sdk';

class LiveKitService {
  createRoomToken(roomName, identity) {
    const at = new AccessToken(process.env.LIVEKIT_API_KEY, process.env.LIVEKIT_API_SECRET, {
      identity,
    });
    at.addGrant({ roomJoin: true, room: roomName, canPublish: true, canSubscribe: true });
    return at.toJwt();
  }
}

export default new LiveKitService();
```

### B. sarvam.service.js

```javascript
// src/services/sarvam.service.js
import axios from 'axios';
import FormData from 'form-data';

const SARVAM_KEY = process.env.SARVAM_API_KEY;
const BASE = 'https://api.sarvam.ai';

class SarvamService {
  // Speech-to-Text
  async transcribe(audioBuffer, language = 'hi-IN') {
    const form = new FormData();
    form.append('file', audioBuffer, { filename: 'audio.wav', contentType: 'audio/wav' });
    form.append('language_code', language);
    form.append('model', 'saaras:v1');
    const { data } = await axios.post(`${BASE}/speech-to-text`, form, {
      headers: { ...form.getHeaders(), 'api-subscription-key': SARVAM_KEY },
    });
    return data.transcript;
  }

  // Text-to-Speech — returns audio Buffer
  async synthesize(text, language = 'hi-IN', speaker = 'meera') {
    const { data } = await axios.post(`${BASE}/text-to-speech`, {
      inputs: [text],
      target_language_code: language,
      speaker,
      model: 'bulbul:v1',
      speech_sample_rate: 16000,
    }, {
      headers: { 'api-subscription-key': SARVAM_KEY },
    });
    return Buffer.from(data.audios[0], 'base64');
  }

  // Intent extraction — mirrors categoryExtractor.js logic but via LLM
  async extractIntent(transcript) {
    const SYSTEM_PROMPT = `You are a service dispatcher for India.
Extract the intent from the user's request (Hindi, English, or code-mixed).
Return ONLY JSON: { "category": string, "description": string, "urgency": "high"|"normal" }
Categories: plumber, electrician, carpenter, painter, cleaner, mechanic, ac_technician, pest_control`;

    const { data } = await axios.post(`${BASE}/chat/completions`, {
      model: 'sarvam-2b',
      messages: [
        { role: 'system', content: SYSTEM_PROMPT },
        { role: 'user', content: transcript },
      ],
      max_tokens: 150,
      temperature: 0.2,
    }, {
      headers: { 'api-subscription-key': SARVAM_KEY },
    });

    try {
      return JSON.parse(data.choices[0].message.content.trim());
    } catch {
      return { category: 'general', description: transcript, urgency: 'normal' };
    }
  }
}

export default new SarvamService();
```

### C. voiceAgent.service.js — Core State Machine

```javascript
// src/services/voiceAgent.service.js
import { Room, RoomEvent } from 'livekit-client';
import livekitService from './livekit.service.js';
import sarvamService from './sarvam.service.js';
import User from '../models/User.model.js';
import Job from '../models/Job.model.js';

class VoiceAgentService {
  async startVoiceSession(roomName, userId, jobId) {
    const token = await livekitService.createRoomToken(roomName, 'node-agent');
    const room = new Room();
    await room.connect(process.env.LIVEKIT_URL, token);

    const user = await User.findById(userId);
    if (!user) {
      await this.speak(room, 'Aapka account nahi mila. Kripya register karo.');
      return room.disconnect();
    }

    if (user.userType === 'client') {
      await this.runClientFlow(room, user);
    } else if (user.userType === 'worker') {
      await this.runWorkerFlow(room, user, jobId);
    }

    await room.disconnect();
  }

  async speak(room, text, lang = 'hi-IN') {
    const audioBuffer = await sarvamService.synthesize(text, lang);
    // Publish audioBuffer to room via LocalAudioTrack
  }

  async listen(room, durationMs = 8000) {
    // Subscribe to remote participant's audio track
    // Collect audio frames for durationMs
    // Return accumulated Buffer
  }

  // --- Client Flow ---
  // Client calls in → describe problem → job created → worker matched
  async runClientFlow(room, user) {
    await this.speak(room, 'Namaste, aapki kya problem hai? Hindi ya English mein batao.');
    const audio = await this.listen(room, 8000);
    const transcript = await sarvamService.transcribe(audio);
    const intent = await sarvamService.extractIntent(transcript);

    // Create job using existing Job model
    const job = await Job.create({
      clientId: user._id,
      category: intent.category,
      description: intent.description,
      location: user.location,
      address: user.address,
      status: 'pending',
    });

    // Find worker using matching logic (50% distance + 30% rating + 20% visibility)
    const workers = await User.find({
      userType: 'worker',
      status: 'available',
      categories: intent.category,
      location: {
        $nearSphere: {
          $geometry: user.location,
          $maxDistance: 10000, // 10 km
        },
      },
    }).limit(1);

    if (workers.length === 0) {
      await this.speak(room, 'Maafi, abhi koi worker available nahi hai. Thodi der baad try karo.');
      return;
    }

    const worker = workers[0];
    job.workerId = worker._id;
    job.status = 'assigned';
    job.assignedAt = new Date();
    await job.save();

    await this.speak(room, `Aapka ${intent.category} worker ${worker.name} mil gaya hai. Woh jaldi aayenge.`);
  }

  // --- Worker Flow ---
  // Worker receives call → hears job brief → accepts or declines
  async runWorkerFlow(room, user, jobId) {
    const job = jobId
      ? await Job.findById(jobId).populate('clientId', 'name phone address')
      : await Job.findOne({ workerId: user._id, status: 'assigned' }).populate('clientId', 'name phone address');

    if (!job) {
      await this.speak(room, 'Abhi aapke liye koi kaam nahi hai.');
      return;
    }

    const brief = `Aapke paas ek ${job.category} ka kaam hai. `
      + `Jagah: ${job.address}. `
      + `Payment: ${job.paymentOffer} rupees. `
      + `Accept karoge?`;

    await this.speak(room, brief);
    const audio = await this.listen(room, 5000);
    const transcript = await sarvamService.transcribe(audio);
    const accepted = /haan|yes|accept|ha|theek|okay/i.test(transcript);

    if (accepted) {
      job.status = 'accepted';
      job.acceptedAt = new Date();
      await job.save();
      user.status = 'busy';
      await user.save();
      await this.speak(room, `Kaam accept hua. Client ${job.clientId.name} ka intezaar kar raha hai.`);
    } else {
      job.status = 'pending';
      job.workerId = undefined;
      await job.save();
      user.status = 'available';
      await user.save();
      await this.speak(room, 'Theek hai, kaam decline kiya. Dhanyavaad.');
    }
  }
}

export default new VoiceAgentService();
```
